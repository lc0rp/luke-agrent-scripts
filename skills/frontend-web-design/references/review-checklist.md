# Review Checklist

Use this before handoff.

## Visual Direction

- Is the direction named and coherent?
- Does the interface feel like one product?
- Does anything feel like generic AI UI filler?
- Is the first viewport one composition or a pile of components?

## Hierarchy

- Is the primary action obvious?
- Can a user scan the page quickly?
- Are there too many emphasis levels?
- Are labels and groupings clear?
- On branded pages, is the brand or product unmistakable in the first screen?

## Layout

- Are alignments clean?
- Are sections necessary?
- Is density appropriate for the product?
- Does mobile preserve task flow instead of merely stacking everything?
- Are fixed or floating elements kept out of the way of key content?

## Typography

- Is there a clear type hierarchy?
- Are font choices intentional?
- Is small text still readable?
- Are there more than two typefaces without a strong reason?

## Color

- Are neutrals doing most of the structural work?
- Is accent color used sparingly and meaningfully?
- Is contrast strong enough?
- Are there multiple competing accent colors?

## Components

- Are buttons, inputs, tables, and lists believable?
- Are cards used only where they help?
- Do interactive elements have hover, focus, active, and disabled states?
- Could any card treatment be removed without losing meaning?

## UX States

- Loading state present?
- Empty state present?
- Error state present?
- Success feedback present where needed?
- Are dead links, fake buttons, or `#` placeholders removed or clearly disabled?

## Accessibility

- Keyboard navigation works?
- Focus ring visible?
- Contrast acceptable?
- Motion reduced when appropriate?
- Labels and affordances clear?
- Skip link or equivalent available when page structure warrants it?

## Content Realism

- Any fake metrics, fake charts, or filler panels?
- Any copy that explains the interface instead of serving the product?
- Does the content feel plausible for a real app?
- Does each section have one job: explain, prove, deepen, or convert?

## Strategic Omissions

- Legal links present when the page needs them?
- Back navigation or exit path present where flows can dead-end?
- Form validation and inline errors present where forms exist?
- Helpful 404 or empty-state path considered when relevant?

## Final Question

- Would a human designer or strong frontend engineer believe this UI was made on purpose?
